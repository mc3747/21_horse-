hexo配置的参考文档：https://hexo.io/zh-cn/docs/

1，检查安装环境

  • 检查node.js版本
    node -v

  • 检查git版本
    git --version

  • 如果未安装请参考 [git]: https://sourceforge.net/projects/git-osx-installer/

2，安装hexo

npm install -g hexo-cli

3，初始化建立博客框架

  • 在你的家目录下创建一个blog文件夹
    mkdir blog

  • 进入目录
    cd blog

  • 初始化目录
    hexo init

  • 开启本地服务
    hexo s

  • 本地浏览器验证：http://localhost:4000/

    4，关联github仓库

  • 新建仓库

    • ⚠️：仓库名一定要与用户名一致，后跟.github.io

  • 配置_config.yml文件

    • 本地blog的路径下，找到_config.yml并打开

5，新建目录文件

  • 新建文件:hexo new "hexo搭建个人博客-1"
  • 在source–_post处生成

6，生成页面，提交代码

  • 生成静态页面:hexo g

  • 提交远程代码:hexo d

  • 生成并提交代码合成:hexo g -d

  • 假如报错，提示无法链接到gi或找不到git，运行：
    npm install hexo-deployer-git --save

  • 远程验证：https://mc3747.github.io和本地运行的界面http://localhost:4000一致即可

7,遇到页面显示空白的排查方法

1,本地验证:hexo s,然后再运行本地环境:http://localhost:4000
2,更改vender:手动将主目录的source/vendors目录修改为source/lib，同时修改主题配置文件_config.yml，将里面的_internal:vendors改为_internal:lib，可是试了一下还不行
3,在主目录第一个文件.deploy_git加上一个名为.nojekyll的空文件就好了，格式为.txt就行
4,在主目录下查看public文件夹中的html文件是否为空,
    第一种：缺少相应包文件
        问题情况： hexo g 命令后无相应静态文件index.html生成。
        解决方案见此篇：hexo g无法生成静态文件

    第二种：缺失相应配置文件
        问题情况：生成了index.html，但是此文件大小为0kb，博客首页文章预览正常显示，点开跳转为空白页面。
        解决方案：git bush后运行hexo g --debug，即可在出现的结果中找到相应错误。例如会出现"can’t find xxxxxx"。即说明某个配置文件被引用，但你的文件里并没有这个文件，进行添加即可。

    第三种：缺少.md文件
        问题情况： hexo g 命令后无相应静态文件index.html生成。
        解决方案：查找_post文件夹是否有相应的.md文件。
5,查看npm的版本
    1_使用npm g -d,会报警告"(node:1736) Warning: Accessing non-existent property 'column' of module exports inside circular dependency"
    2_查看当前npm版本:npm -v
    3_安装高级版brew install npm 
    4_安装指定稳定版brew install node@12
    (
     步骤不可用:3_查看稳定版:sudo n --stable
     4_管理npm:sudo npm i -g n
     5_更换npm为稳定版: sudo n stable
    )
6,使用默认主题
7,更新hexo
    查看当前hexo的版本:npx hexo -v
    更新npm:npm update
    更新后查看最新的版本:npx hexo -v
8,更新next主题:进入next主题目录,git pull
