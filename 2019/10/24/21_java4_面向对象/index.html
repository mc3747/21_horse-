6,方法

6.1 方法的种类

2，普通方法

6.2 方法的注意项

1，方法的重载：
    必须拥有不同的参数列表
2，可变参数：
    以下2种表示方法
    void function(String... args);
    void function(String [] args);
3，finalize() 方法：
    在对象被垃圾收集器析构(回收)之前调用，这个方法叫做 finalize( )，它用来清除回收对象

7,this关键字

7.1. this作用:

1. 访问当前类中的成员变量,普通方法,构造方法
2. 只能在构造方法(普通方法中不行🚫)中,使用this调用其他的构造方法
3. 构造方法调用构造方法,this语句要放在第一句

7.2. 本质

隐藏的,位置最为靠前的一个参数

8 super关键字

2. 调用父类的方法(包括构造方法)

9，继承(Inheritance)

1. 重载和重写
    1. 方法签名(method signature):方法名 和 参数类型
    2. 重载(overload):方法名相同,方法签名不同(要么参数个数,要么参数类型不同)
    3. 重写(override):方法签名相同,子类重写父类
2. 结构:extends
3. 基类: java.lang.Object
4. 成员变量:
    1. 名称可以相同(通过this和super调用,不推荐)
    2. 继承后,子类包含父类的所有成员变量(包括private修饰的,可以用set和get方法访问)
5. 方法的重写
    1. 子类的实例方法签名和父类一样,才叫方法的重写
    2. 子类的类方法签名(静态)和父类一样,不叫方法重写
    3. 子类override的方法权限必须 >= 父类
    4. 子类override的返回值范围 <= 父类

10 访问控制

1. 4个访问权限
    1. public
    2. protected
    3. 无修饰
    4. private:仅自己类中可见
2. 访问权限修饰的类型
    1. 成员变量
    2. 方法
    3. 嵌套类
3. 注意
    1. 只有public和无修饰,可以用在顶层类(top-level Class)
    2. 均不可以修饰局部类,局部变量
    3. 顶层类可以多个,public修饰的顶层类,必须和类名一样

11 注解和注释：

1. 注释Comment
2. 注解Annotation
    1. 编译器特性
    2. @Override:告诉这个是重写后的方法
    3. @SupperessWarnings("unused"):删除警告
    4. @Deprecated:表示内容过期,不推荐使用

12 封装

1. 成员变量private化
2. 提供public的getter和setter方法

13 static

1. 成员变量
    1. 未被static修饰 ->实例变量
        1. 每个实例内部都有一份内存
        2. 只能实例访问,不能类访问
    2. 被static修饰 ->静态变量(类变量)
        1. 只占用一份固定内存(存储在方法区)
        2. 可以通过类访问(推荐),也可以用实例访问(不推荐)
    3. 同一个类中,不能有同名的实例变量和类变量
2. 方法
    1. 未被static修饰 ->实例方法
        1. 外部只能通过实例访问,不可以通过类访问
        2. 内部可以使用this
        3. 内部可以直接访问实例变量,实例方法
        4. 内部可以直接访问类变量,类方法
    2. 被static修饰 -> 静态方法(类方法)
        1. 外部可以通过实例和类访问
        2. 内部不可以使用this
        3. 内部可以直接访问实例变量,实例方法
        4. 内部不可以直接访问类变量,类方法
    3. 同一个类中,不能有相同签名的实例方法和类方法
3. 静态导入
    1. 格式:import static com.baidu.mc.model.*
    2. 作用:
        1. 可以省略类名来访问静态成员(成员变量,方法,嵌套类)
        2. 消除重复的类名,提高可读性
        3. 防止过度地静态导入,以至于分不清是哪个类中定义
    3. 案例:import static java.lang.Math.PI

14 成员变量的初始化

 1. 编译器自动初始化默认值
 2. 手动给示例变量初始化
    1. 声明中
    2. 在构造方法中
    3. 在初始化块中(每创建一个示例,都有一个初始化块,都放在构造方法的第一行)
1. 手动给静态类变量初始化
    1. 声明中
    2. 在构造方法中
    3. 在静态初始化块中(当第一个类初始化时,会执行静态初始化块;)

15 单例模式

1. 恶汉式(建议)
1. 懒汉式(有线程问题,不建议)

16 final

1. 修饰子类:不能被子类化,不能被继承
2. 修饰变量:只能进行一次赋值
3. 修饰方法:不能被重写
4.

17 常量constant

1. 通常使用static和final配合
2. 只能修饰基本类型和字符串类型
3. 在编译时就能确定值,也叫编译时常量(类似于C语言的宏)
4. 案例:
    1. public static final double PI = 3.1415926;
    2. private static final int NOT_FOUND = -404;
