108_网络1_基本概念

发表于 2020-04-14 | 分类于 计算机基础 | | 阅读次数:

1. OSI模型

  1. 物理层：
    1_将信息编码成电流脉冲或其它信号用于网上传输；
    2_为数据链路层提供物理供应,传递透明的比特流;
    3_设备有同轴电缆,双绞线,网线,调制解调器,中继器,集线器等

  2. 数据链路层:
    1_指定拓扑结构并提供硬件寻址。即规定了0和1的分包形式，确定了网络数据包的形式；
    2_此层将数据分帧,处理流控制;
    3_常见的设备有网卡,网桥,交换机

  3. 网络层
    1_通过寻址来建立两个节点之间的连接,源和终点之间建立连接;
    3_此层的数据叫包;
    2_IP协议在该层(IPV4,IPV6)

  4. 传输层：
    1_传输层向高层􏰁提供可靠的端到端的网络数据流服务。每一个应用程序都会在网卡注册一个端口号，该层就是端口与端口的通信
    2_此层的数据叫段;
    3_TCP协议在该层

  5. 会话层：
    1_会话层建立、管理和终止表示层与实体之间的通信会话；
    2_建立一个连接（自动的手机信息、自动的网络寻址）;

  6. 表示层:
    1_表示层􏰁供多种功能用于应用层数据编码和转化,以确保以一个系统应用层发送的信息 可以被另一个系统应用层识别;可以理解为：解决不同系统之间的通信，eg：Linux下的QQ和Windows下的QQ可以通信;
    2_包括数据格式交换,数据的加解密,数据的压缩和类型转换

  7. 应用层:
    1_OSI 的应用层协议包括文件的传输、访问及管理协议(FTAM) ,以及文件虚拟终端协议(VIP)和公用管理系统信息(CMIP)等;
    2_包括http,https,FTP,SSH等

  8. 注意点:

    1. 第57层(上三层)为应用层,用软件控制;14(下四层)为数据流程,管理硬件;
    2. 除了物理层,其他都是用软件实现
    3. 发送数据时,从第七层传到第一层;接受数据时相反
    4. 数据在不同的层会被拆分,在物理层叫比特流,在数据链路层叫帧,在网络层叫包,在传输层叫段,这种叫法为PDU(协议数据单元)

2. TCP相关概念

Transmission Control Protocol/Internet Protocol,传输控制协议/因特网互联协议，又名网络通讯协议，是Internet最基本的协议、Internet国际互联网络的基础

2.1 TCP协议设计原理

TCP/IP 的设计,是吸取了分层模型的精华思想——封装。每层对上一层􏰁供服务的时 候,上一层的数据结构是黑盒,直接作为本层的数据,而不需要关心上一层协议的任何细节。

1_网络接口层:
    网络接口层包括用于协作IP数据在已有网络介质上传输的协议。
    它定义像地址解析协议(Address Resolution Protocol,ARP)这样的协议,􏰁供 TCP/IP 协议的数据结构和实际物理硬件之间的接口。
    可以理解为：确定了网络数据包的形式。

2_网间层
    网间层对应于 OSI 七层参考模型的网络层，本层包含 IP 协议、RIP 协议(Routing Information Protocol,路由信息协议),负责数据的包装、寻址和路由。同时还包含网间控制报文协议(Internet Control Message Protocol,ICMP)用来􏰁供网络诊断信息；可以理解为：该层时确定计算机的位置。

3_传输层
传输层对应于 OSI 七层参考模型的传输层,它􏰁供两种端到端的通信服务。其中 TCP 协议(Transmission Control Protocol)􏰁供可靠的数据流运输服务,UDP 协议(Use Datagram Protocol)􏰁供不可靠的用户数据报服务。
TCP:三次握手、四次挥手;UDP:只发不管别人收不收得到--任性哈

4_应用层
应用层对应于 OSI 七层参考模型的应用层和表达层；
