1. 基本控件

1.1 背景图片

1. 使用view的background-image属性
    1. src不能使用本地绝对路径和相对路径
    2. 可以使用网络图片
    3. 可以将图片转成base64来使用
2. 使用image标签
    1. 为单标签
    2. 可以使用相对路径,绝对路径,网络路径

1.2 按钮

3. 使用button标签,一般在表达中使用,属于重量型
4. 使用普通view标签的事件绑定,bindtap="方法名"
5. 按钮点击响应同一事件的区分
    1. 方法1:传入js文件中的数组,使用默认的item和index区分
    2. 方法2:直接使用js文件中传入的字段来区分
    3. 方法3:传入数组数据:<view wx:for="{{ [88, 77, 55, 44, 33] }}">
    4. 最后在响应方法begin: function(evt) 中区分evt.currentTarget.dataset.index

1.3 循环

1. 循环绑定数组
2. 循环指定元素的名称
3. 循环指定下标的变量名称:

1.4 数据绑定

1. mustache语法
2. 双大括号引用

1.5 隐藏元素

1. 方法1:visibulity属性设置为hidden(占位,隐藏)
2. 方法2:display属性设置为none(不占位,但渲染,尺寸为0)
3. 方法3:微信中的条件渲染
    1. 使用if:<view wx:if="{{condition}}"> True </view>
    2. 使用if--else:<view wx:if="{{length > 5}}"> 1 </view>
        <view wx:elif="{{length > 2}}"> 2 </view>
        <view wx:else> 3 </view>

1.6 字体单位

1. px:像素
2. em:（font size of the element）相对于父元素的值
3. rem:（font size of the root element）根元素的字体
4. %：相对单位，每个子元素透过「百分比」乘以父元素的px值
5. 任意浏览器的默认字体高都是16px。所有未经调整的浏览器都符合: 1em=16px
6. smaller：约为父层的80%; larger：约为父层的120%
7. 微信小程序中:rpx（responsive pixel)
    1. 可以根据屏幕宽度进行自适应。规定屏幕宽为750rpx。如在 iPhone6 上，屏幕宽度为375px，共有750个物理像素，则750rpx = 375px = 750物理像素，1rpx = 0.5px = 1物理像素

1.7 固定底部文字

1. 方案1:使用绝对布局
    1. 子绝父相,相对的是整个屏幕,如果距离太大,可能会跟上部数据重合
2. 方案2:直接相对上部元素
    1. 使用margin-top,会跟随上部元素,不会固定在最底部
3. 方案3:使用浮动
4. 方案4:flex布局,使用两端对齐(推荐)
