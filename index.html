3.提交时.DS_Store和.xcuserstate文件冲突

1.1 一个仓库的解决办法:
    1. 删除本地的.DS_Store文件:find . -name '*.DS_Store' -type f -delete
    2. 删除本地的.xcuserstate文件:find . -name '*.xcuserstate' -type f -delete
    3. 新建一个.gitignore文件,里边的内容为两行:*.DS_Store和*.xcuserstate

1.2 全局的结局解决办法
    1.找到全局的gitconfig文件:目录~/.gitconfig,
    2.在同目录下,新建一个~/.gitignore_global文件,文件内容和.gitignore文件一样
    3..gitconfig引入.gitignore_global文件:
        方法1:直接在文件里边改:excludesfile = /Users/reon/.gitignore_global **
        方法2:直接使用命令行:git config --global core.excludesfile/Users/reon/.gitignore_global命令来实现

100_加密相关1_编码

发表于 2020-03-20 | 分类于 计算机基础 | | 阅读次数:
字数统计: 300 | 阅读时长 ≈ 1

1，ASCII码

1,American Standard Code for Information Interchange
2,ASCII码：1个字节;整型：至少2个字节
3,在网页中使用应在数字串前后分别加上“&#”和“;”，例如数字0的ASCII编码是48，则在网页中使用的格式应为“&#48;”。符号'#'的ASCII编码是35,则在网页中使用的格式应为“&#35;”
4,转换工具地址：https://www.qqxiuzi.cn/bianma/ascii.htm

2，Unicode：

1,只是一个符号集，包含所有的符号
2,以'\u'开头跟4位
3,例如字符'马'的Unicode编码'\u9a6c'

3，UTF-8

1,UTF-8 是 Unicode 的实现方式之一
2,存储的内容,字节可变
3,以&#开头的后接十进制数字，以&#x开头的后接十六进制数字
4,例如字符'马'的UTF-8编码为'&#x9A6C;'

4，url编码：

1,就是一个字符ascii码的十六进制。不过稍微有些变动，需要在前面加上“%”
2,例如字符'马'的UTF-8编码为'%u9A6C'

5，base64

1,用64个字符来表示任意二进制数据的方法
2,例如字符'马'的base64编码为'6ams'

3 沙盒目录:3个文件夹

1,Documents：苹果建议将程序中建立的或在程序中浏览到的文件数据保存在该目录下，iTunes备份和恢复的时候会包括此目录
2,Library：存储程序的默认设置或其它状态信息；
         2.1 Library/Caches：存放缓存文件，iTunes不会备份此目录，此目录下文件不会在应用退出删除.
         2.2 Library/preferences: 存放的是 user default 存储的信息，iTunes会备份此目录， 应用程序重新启动不会丢弃数据，我们使用 NSUserDefaults写的设置数据都会保存到该目录下的一个plist文件中，这就是所谓的写到plist中！
3,tmp：提供一个即时创建临时文件的地方， iTunes不会备份此目录
