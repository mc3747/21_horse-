1，怎样评估算法的好坏

1，正确性
2，可读性
3，健壮性
4，时间复杂度：time complexity，估算程序的执行次数
5，空间复杂度：space ciplexity，估算所需占用的存储空间

2，评估算法的方法

  1. 事后统计法

    ✅优点：准确
❎缺点：
    1 执行时间严重依赖硬件以及运行时各种不确定的环境因素 
    2 必须编写相应的测算代码 
    3 测试数据的选择比较难保证公正性

  2. 大o统计法

    ✅优点：简单估算
    ❎缺点：不特别准确,实际对应的是汇编的操作数

3，大o统计法案例

    // O(1)
    public static void test1(int n) {
        // 1次，O(1)
        if (n > 10) { 
            System.out.println("n > 10");
        } else if (n > 5) { // 2
            System.out.println("n > 5");
        } else {
            System.out.println("n <= 5"); 
        }

        // 1 + 4 + 4 + 4
        for (int i = 0; i < 4; i++) {
            System.out.println("test");
        }

    }

    // O(n)
    public static void test2(int n) {

        // 1 + 3n
        for (int i = 0; i < n; i++) {
            System.out.println("test");
        }
    }

    // O(n^2)
    public static void test3(int n) {
        // 1 + 2n + n * (1 + 3n)
        // 1 + 2n + n + 3n^2
        // 3n^2 + 3n + 1


        // O(n)
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < n; j++) {
                System.out.println("test");
            }
        }
    }

    // O(n)
    public static void test4(int n) {
        // 1 + 2n + n * (1 + 45)
        // 1 + 2n + 46n
        // 48n + 1
        for (int i = 0; i < n; i++) {
            for (int j = 0; j < 15; j++) {
                System.out.println("test");
            }
        }
    }

    // O(logn)
    public static void test5(int n) {
        // 8 = 2^3
        // 16 = 2^4

        // 3 = log2(8)
        // 4 = log2(16)

        // 执行次数 = log2(n)
        while ((n = n / 2) > 0) {
            System.out.println("test");
        }
    }

    // O(logn)
    public static void test6(int n) {
        // log5(n)
        while ((n = n / 5) > 0) {
            System.out.println("test");
        }
    }

    // O(nlogn)
    public static void test7(int n) {
        // 1 + 2*log2(n) + log2(n) * (1 + 3n)
        // 1 + 3*log2(n) + 2 * nlog2(n)

        for (int i = 1; i < n; i = i * 2) {
            // 1 + 3n
            for (int j = 0; j < n; j++) {
                System.out.println("test");
            }
        }
    }

//    斐波那契数2：O(n)
    public static int test9(int n) {
        if (n <= 1) return n;
        int first = 0;
        int second = 1;
        for (int i = 0; i < n - 1; i++) {
            int sum = first + second;
            first = second;
            second = sum;
        }
        return second;

    }

        int a = 10;

        int[] array = new int[n];
        for (int i = 0; i < array.length; i++) {
            System.out.println(array[i] + c);

    }

4，大o表示法总结

  1. 大O只是是估算
  2. 大O也可以表示空间复杂度
  3. 对数阶的公式：log2(n) = log2(9) * log9(n)
  4. 常数阶O(1)，对数阶O(log2n)，线性阶O(n)，线性对数阶O(nlog2n)，平方阶O(n2)，立方阶O(n3)，…， k次方阶O(nk)，指数阶O(2n)。随着问题规模n的不断增大，上述时间复杂度不断增大，算法的执行效率越低。
  5. 一般规律：O(1) < O(log2n) < O(n) < O(n*log2n) < O(n2) < O(n3) < O(2n) < O(3n) < O(n!) （c是一个常量）

3，生成.ipa文件

1. 生成.app文件

1. 方法1：编译到模拟器或者真机后的.app文件 -》 showInFinder

2. 方法2：生成.app文件
    ①选中设备Generic iOS Devices，
    ②xcode默认archive生成的是release模式，需要将服务器地址更正一下(在代码中改正)；
    ③然后Xcode菜单栏->Product->Archive；
    ④生成的xcproject文件 -》showInFinder
    ⑤复制后更改xcproject文件的后缀为.ipa,形成文件夹,进入文件夹找到.app文件

3. 方法3：直接进目录 ~/Library/Developer/Xcode/Archives查找方法2中生成的.xcproject文件

2. 生成.ipa文件

1. 方法1：新建Payload文件夹生成
    ①: 新建“Payload”文件夹，注意名字要一字不差；
    ②: 将你的.app包放到Payload中，注意app的名字不做任何更改，就用xcode生成的app名称；
    ③: 在Payload文件夹上右键压缩成zip，然后将生成的.zip文件后缀改成.ipa即可

2.     方法2：脚本生成

①：进入脚本文件make_ipa_file.sh目录，        ②：执行脚本语句：sh make_ipa_file.sh(也可以把 make_ipa_file.sh 改为 make_ipa_file_sh 名字，然后在终端为文件添加可执行权限：chmod +x /Users/HTC/Desktop/make_ipa_file_sh,这样就生成了 Unix 文件，双击就会打开到终端)
③：按照提示，分别输入.app文件目录，.ipa文件保存目录（默认桌面），生成的ipa文件名称
④：脚本链接： <https://github.com/iHTCboy/iShell>，代码如下
