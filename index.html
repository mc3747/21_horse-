2 获取设备的device token

1. 含义: iOS 设备向 APNs 注册推送通知服务时，返回的设备标识符
2. 获取方法:
进入app delegate的代理方法- (void)application:(UIApplication *)application didRegisterForRemoteNotificationsWithDeviceToken:(NSData *)deviceToken,获取回调

1,利用smartPush工具
2,选择推送证书,填入device token
3,先点连接服务器,再直接推送
4,在手机终端上观察

4 第三方:个推

1,配置证书:
    1,将钥匙串中的推送证书,转成p12
    2,进入个推界面,选择对应的环境 ->配置管理 ->应用配置
    3,替换p12文件(注意环境)
    4,可以测试证书推送

2,iOS进入透传消息:
    1,填写各个平台
    2,进入app代码中,个推api的- (void)GeTuiSdkDidRegisterClient:(NSString *)clientId方法中获取cid
    3,填写完标题内容,可以预览,前提是app要启动一次
    4,发送推送

2. 生成ipa文件的方式

2.1. 方法1:product -> archive(其中选中设备Generic iOS Devices)

1. Step1:生成xcproject文件；（文件路径：~/Library/Developer/Xcode/Archives）
2. Step2:更改后缀成ipa，形成文件夹,进入文件夹找到.app文件；
3. Step3: 新建“Payload”文件夹，注意名字要一字不差；
4. Step4: 将你的.app包放到Payload中，注意app的名字不做任何更改，就用xcode生成的app名称；
5. Step5: 在Payload文件夹上右键压缩成zip，然后将生成的.zip文件后缀改成.ipa即可

2.2. 方法2:终端命令

    1. 准备ExportOptions.plist的配置文件

    2. 进入工程目录,清理工程:xcodebuild clean

    3. 使用下面的命令archive导出.xcarchive文件:
        xcodebuild archive -project TestAutoPacking.xcodeproj -scheme TestAutoPacking -archivePath /dandy/xmeAutoArchive/TestAutoPacking.xcarchive
或者:
        xcodebuild archive -workspace TestAutoPacking.xcworkspace -scheme TestAutoPacking -archivePath /dandy/xmeAutoArchive/TestAutoPacking.xcarchive

    4. 导出ipa包
        xcodebuild -exportArchive -archivePath /archive文件路径/archive文件名称.xcarchive -exportOptionsPlist /ExportOptions文件路径/ExportOptions.plist -exportPath /ipa导出的文件路径/

2.3 方法3:使用脚本生成ipa文件

2.4 方法4:利用jenkens工具生成ipa文件

2.5 方法5:利用fastlane工具生成ipa文件

3 生成的ipa文件怎么安装到手机

3.1 方法1:百度网盘

用户下载得到.ipa文件(例如放大百度网盘等),通过itools,或者Apple Configurator安装到手机

3.2 方法2:服务器链接

1. 开发者账号

1. 个人($99,可以发布APP Store,申请资料少)
2. 公司($99,可以发布APP Store,申请资料多)
3. 企业($299,不可以发布APP Store,申请资料多)

2. 证书

1. 本地证书
2. 开发者证书-development
3. 发布者证书-production
4. 推送证书-push

3. 配置文件(provision profile,也叫pp文件)

1. 开发:development
2. 发布:ad hoc
3. 发布:app store

4. p12证书

1. 进入钥匙串->展开某个证书->两者选中右键导出
2. 自己设置一个密码
3. 作用:
    1. 一般是账号里的每个证书,对应一台电脑(除非有原始证书的ca文件)
    2. 最多只能建立3个开发证书,3个发布证书,超过就要生成p12证书

5.技巧

1. 给其他开发者p12文件和描述文件就可以了,p12文件让电脑就有开发的证明,描述文件让xcode识别合法性
2. 通过svn将p12文件,和描述文件共享,及时更新
