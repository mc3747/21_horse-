1 概念：

px：pixel，像素，电子屏幕上组成一幅图画或照片的最基本单元
pt: point，点，印刷行业常用单位，等于1/72英寸
ppi: pixel per inch，每英寸像素数，该值越高，则屏幕越细腻
dpi: dot per inch，每英寸多少点，该值越高，则图片越细腻
dp: dip，Density-independent pixel, 是安卓开发用的长度单位，1dp表示在屏幕像素点密度为160ppi时1px长度
sp: scale-independent pixel，安卓开发用的字体大小单位。

2 根据屏幕分辨率，求ppi

ppi= 屏幕对角线上的像素点数/对角线长度 = √ （屏幕横向像素点^2 + 屏幕纵向像素点^2）/对角线长度

3 查看硬件设备的网站：

https://www.mydevice.io，用终端登陆后，就可以知道该终端的尺寸信息

1，元素的属性

全局属性参考：<https://www.w3cschool.cn/htmltags/ref-standardattributes.html>

1.1 常见的全局属性：

1，id属性
    用于html元素的唯一标识，css使用className
    url使用锚点访问：https://foo.com/index.html#bar，滚动到id为bar的元素的位置
2，class属性
    表示具有相同属性
    不唯一
    可以叠加，中间用空格
3，title属性
    鼠标悬浮，显示的title
4，tabindex
    表示用户按下 Tab 键的时候，网页焦点转移的顺序
5, accesskey
    1，网页元素获得焦点的快捷键
    2，例如：<button accesskey="s">提交</button>
    3，Chrome 浏览器在 Windows 系统和 Linux 系统的快捷键是Alt + 字符键，在 Mac 系统的快捷键是Ctrl + Alt + 字符键
    4，accesskey如果跟操作系统或浏览器级别的快捷键有冲突，这时不会生效
6，hidden
    1，布尔属性，表示当前的网页元素不再跟页面相关，因此浏览器不会渲染这个元素，所以就不会在网页中看到它
    2，<p hidden>本句不会显示在页面上。</p>
    3，CSS 的可见性设置，高于hidden属性。如果 CSS 设为该元素可见，hidden属性将无效
7，lang
    1，网页元素使用的语言
    2，类别
        zh：中文
        zh-Hans：简体中文
        zh-Hant：繁体中文
        en：英语
8，dir
    1，表示文字的阅读方向
    2，可选值
        ltr：从左到右阅读，比如英语。
        rtl：从右到左阅读，阿拉伯语、波斯语、希伯来语都属于这一类。
        auto：浏览器根据内容的解析结果，自行决定
9，contenteditable
    1，允许用户修改内容
    2，可选值
        true或空字符串：内容可以编辑
        false：不可以编辑
10，spellcheck
    1，拼写检查
    2，可选值
11，data-*
    *可以被取代，为自定义属性，用来传参等
12,    style
    元素的 CSS 样式
13,事件处理属性
    见下边第三部分

    表示一个可选的断行。如果一行的宽度足够，则不断行；如果宽度不够，需要断行;

    不推荐使用<i>，Italic

    <sub>标签将内容变为下标，bottom
    <sup>标签将内容变为上标，peak
    <var>标签表示代码或数学公式的变量
<u>，<s>：下划线和删除线
    <u>会产生下划线：underline
    <s>标签是一个行内元素，为内容加上删除线：strike

2，百度

1，精准搜索，加上引号“”，
    例子“香蕉牛奶”
2，添加书名号《》，一是书名号会出现在搜索结果中；二是被书名号扩起来的内容，不会被拆分
    例子:《香蕉牛奶》
3，包含特定查询词“+”
    例子:厨子戏子 +厨子
4，排除关键词“-”
    例子:厨子戏子 -痞子
5，百度高级搜索：<http://www.baidu.com/gaoji/advanced.html>
6，把搜索范围限定在网页标题中——intitle:标题
    例子：iOS intitle:零基础
7，把搜索范围限定在特定站点中——site:站名
    例子：五月天 site:www.dytt8.net
8，把搜索范围限定在[url]url链接中——inurl:链接
    例子:inurl:admin
9，Filetype搜索范围限定在指定文档格式中
    例子:中国电信 filetype:doc

3，谷歌：

1，逻辑与：空格表示
    例子：苹果 癌症
2，逻辑或：OR表示
    例子：apple ORmother
3，site：可以查询某个网站页面的收录情况，
    例子：apple site:http://www.apple.com/查看包含关键词apple的网页收录情况
4，link: 查询所有和某个URL有链接的网页，只能单独使用
    例子：link:http://www.apple.com/
5，inurl：
    apple inurl:happy返回结果为包含happy的URL，其它任何地方则包含了apple
6，intitle：
    apple intitle:happy返回结果为包含happy的标题，其它任何地方则包含了apple
7，intext：
    apple intitle:happy返回结果为包含happy的正文，其它任何地方则包含了apple    
8，allinurl:查询返回的网页链接中包含所有查询的关键词
9，allintitle：查询返回的网页标题中包含所有查询的关键词

68_方法论2_如何掌握所有的编程语言

发表于 2019-12-30 | 分类于 其他 | | 阅读次数:
字数统计: 1.3k | 阅读时长 ≈ 4

1，我所理解的编程语言

1.1，语言的大致分类：

机器，汇编，高级

1.2，面向对象和面向过程：

1,最本质的区别在于考虑问题的出发点不同，面向过程是以事件流程为考虑问题的出发点，而面向对象则是以参与事件的角色（对象）为考虑问题的出发点，所以面向对象在处理问题时更加灵活。
2,目前，面向过程的语言更多被用于处理底层业务，而面向对象编程则更多用于实现一些业务逻辑复杂的大型系统
3,从结构上来说，面向过程的特点是模块化和流程化，而面向对象的特点是封装、继承和多态，这里面就有本质的区别了。
4,面向过程以功能（函数）作为模块化的基础，而面向对象则以类作为功能的组织基础

1.3，解释语言和编译语言：

定义：
    1，编程语言是把程序员所定义的代码翻译成计算机所认识的二进制代码的工具；
    2，脚本语言是解释执行的（如写的前端代码过能浏览器这个解释器，把代码呈现出web界面，这就是在运用脚本语言，进行编程；
区别：
    1，脚本语言因为不需要编译器，省去了编译的过程，这就极大的减少了，开发的的时间，而编程语言，因为需要编译，所以可能需要的时间更加长一些，
    2，脚本语言是一种动态语言，也就是说可以实时的更改代码，而不需要将程序停止下来，这是一种高级特性，而Java等编程语言，是静态的语言，一旦编译完成并且运行，就不能更改代码，除非将程序停止下来，但是这样的代价是比较昂贵的。
    3，脚本语言非常容易学习，但是也造成了它的不足，就是不全面，缺乏系统性，语法比较散漫，而高级编程语言，虽然相对难学，但是规则强，可以编程出，简洁美观的代码，并且可读性也相对较强。
    4，一般来说脚本语言通用性较差，但是可以通过专门的应用来调整。
    5，随着技术的发展，其实脚本语言变得越来越强，和编程语言的界限也比较模糊，比如Python，可以将它视为编程语言了，因为它很强大。

1.4，误区：

1，刻意地贬低某类语言；
2，人云亦云地跟风或者吹捧某类语言；
3，流于表面地去评价某类语言难，语言流行，而不去深入思索

2，类比例子：选电脑

一个程序语言就像一台电脑。它的牌子可能叫“联想”，或者“IBM”，或者“Dell”，或者“苹果”。那么，你可以说苹果一定比 IBM 好吗？你不能。你得看看它里面装的是什么型号的处理器，有多少个核，主频多少，有多少 L1 cache，L2 cache……，有多少内存和硬盘，显示器分辨率有多大，显卡是什么 GPU，网卡速度，等等各种“配置”。有时候你还得看各个组件之间的兼容性。

3，方法

1，通用语言特性有哪些
2，特有语言特性有哪些
3，动手实现语言特性
    类型推导，类型检查，惰性求值等的实现
4，入门语言选哪些
    选java，c，js，python，
    不推荐shell，Go，php
5，高手怎样接触新语言
    1，确定好语言
    2，熟悉工具，搭建好环境
    3，理清通用语言特性和特有语言特性
    4，为实现某一个效果，逐渐带出问题，然后查阅，逐步实现自己的想法
    5，不要从书籍或者手册开始，要掌握关键，忽略次要

4，通用语言特性

变量定义
算术运算
for 循环语句，while 循环语句
函数定义，函数调用
递归
静态类型系统
类型推导
lambda 函数
面向对象
垃圾回收
指针算术
goto 语句    
......
  ⚠️   如果你不能用一种语言里面的基本特性写出好的代码，那你换成另外一种语言也无济于事。你会写出一样差的代码。我经常看到有些人 Java 代码写得相当乱，相当糟糕，却骂 Java 不好，雄心勃勃要换用 Go 语言。
　　这些人没有明白，是否能写出好的代码在于人，而不在于语言。如果你的心中没有清晰简单的思维模型，你用任何语言表述出来都是一堆乱麻。如果你 Java 代码写得很糟糕，那么你写 Go 语言代码也会一样糟糕，甚至更差。
　

5,总结：

1，学号编程语言的根本，是重视语言特性，而不是语言
2，任何语言在我的面前，都是可以被任意拆卸组装的玩具，而不再是凌驾于我之上的神圣。
