4. Step4: 将你的.app包放到Payload中，注意app的名字不做任何更改，就用xcode生成的app名称；
5. Step5: 在Payload文件夹上右键压缩成zip，然后将生成的.zip文件后缀改成.ipa即可

        xcodebuild archive -project TestAutoPacking.xcodeproj -scheme TestAutoPacking -archivePath /dandy/xmeAutoArchive/TestAutoPacking.xcarchive
或者:
        xcodebuild archive -workspace TestAutoPacking.xcworkspace -scheme TestAutoPacking -archivePath /dandy/xmeAutoArchive/TestAutoPacking.xcarchive

    4. 导出ipa包
        xcodebuild -exportArchive -archivePath /archive文件路径/archive文件名称.xcarchive -exportOptionsPlist /ExportOptions文件路径/ExportOptions.plist -exportPath /ipa导出的文件路径/

95_iOS18_苹果证书相关

发表于 2020-03-07 | 分类于 iOS | | 阅读次数:
字数统计: 718 | 阅读时长 ≈ 2

1. 开发者账号

1. 个人($99,可以发布APP Store,申请资料少)
2. 公司($99,可以发布APP Store,申请资料多)
3. 企业($299,不可以发布APP Store,申请资料多)

2. 标识符:Identifiers

1. Explicit App ID ：唯一的App ID，用于唯一标识一个应用程序
2. Wildcard App ID：含有通配符的App ID，用于标识一组应程序(使用这种id的应用不能使用通知功能,不常用)
3. 1个bundle ID <-> 一个证书 <->一个配置文件

3. 证书:Certificates

1. 本地证书
2. 开发者证书-development
3. 发布者证书-production
4. 推送证书-push

4. 配置文件(provision profile,也叫pp文件)

1. 开发:development
2. 发布:ad hoc
3. 发布:app store

5. p12证书

1. 进入钥匙串->展开某个证书->右键导出,默认为p12(或者两者选中右键导出)
2. 自己设置一个密码
3. 作用:
    1. 一般是账号里的每个证书,对应一台电脑(除非有原始证书的ca文件)
    2. 最多只能建立3个开发证书,3个发布证书,超过就要生成p12证书

6. 设备 Devices

1_添加设备后,
2_重新下载一下配置文件
3_删除原配置文件
4_安装新配置文件

7.技巧

1. 给其他开发者p12文件和描述文件就可以了,p12文件让电脑就有开发的证明,描述文件让xcode识别合法性
2. 通过svn将p12文件,和描述文件共享,及时更新
3. 已经用旧的证书打包好的ipa,如果删掉打包此ipa的证书,那么该版本的ipa将用不了,谨记⚠️

6. 问题

  1. 生成证书请求文件(CSR),新建证书时需要

    1. 新建CSR证书
    2. 填上名称地址后->存储到本地磁盘
    3. 钥匙串会多一对公钥和私钥
    4. 一个CSR文件可以去新建各种证书(开发的,发布的)

  2. 问题1:根据CSR证书,新建各种证书

    1. 入iOS Dev Cneter，点击”Certificate” - “All”，点击右上角”+”
    2. 配置各种证书
    3. 选择CSR证书

  3. 问题2:证书过期了怎么办

    1. 直接在证书里边修改
    2. 再下载生成,安装本地
    3. 导出p12给别人

  4. 问题3:根据证书新建配置文件

    1. 一定要选择app id(由id名称 + 开发者账号.bundleid构成)
    2. 配置文件要选择对应的证书

  5. 问题4:新建p12文件

  6. 问题5:证书不信任,需要重新配置本地csr文件,然后新建证书

  7. xcode中配置不同环境的证书
    报错类似:xxx is automatically signed for development, but a conflicting code signing identity iPhone Distribution has been manually specified. Set the code signing identity value to "iPhone Developer" in the build settings editor, or switch to manual signing in the project editor
    解决:设置sign code(一定要配置正确)
